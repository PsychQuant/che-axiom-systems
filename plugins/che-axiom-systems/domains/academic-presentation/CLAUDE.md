# Academic Presentation — 學術報告公理

學術會議 oral 報告（投影片＋講稿）的建構與交付原則，以 ASBE 格式公理化。針對混合背景聽眾、常見的非母語講者、以及硬性時間 slot 的情境。

**Source**：2026-07-18 從 IMPS 2026 oral 投影片製作（`measurement_paradigm_unification/article2_joint_thurstonian`）與貴德街預演回饋（雷庚玲、楊昊紘、李琴）萃取，並參照博士口試 deck 慣例。

## 公理總覽

| id | 名稱 | 一句話 |
|----|------|--------|
| A1 | Example-First Grounding | 先給具體例子讓聽眾進入，再進抽象；開場就讓 study question 可見 |
| A2 | Single Channel per Moment | 不要逼聽眾同時讀密集文字又聽不同的話（mental load 超載） |
| A3 | Source-of-Truth Fidelity | 投影片上每個數字／術語／引用都追溯到權威來源、核對而非憑記憶 |
| A4 | Plain Register | 素名詞短語標題＋完整平實句；不用散文式 tagline、小聰明成語、破折號鏈、分號串句（AI tell） |
| A5 | Navigational Scaffolding | 前面放 outline、每頁放位置標記，聽眾隨時知道「講到哪」 |
| A6 | Figure over Dense Table | 關係型資訊優先用圖／流程圖，不用密集表格 |
| A7 | Cross-Audience Term Grounding | 混合聽眾就地定義術語；每個 claim 引用正確的原始出處 |
| A8 | Honest Result Framing | 如實報告 null 結果；框定結果能與不能說明什麼，不誇大 |
| A9 | Time-Budget Discipline | 講稿寫到 slot 內留 buffer，長度用「量測」（字數／讀秒）決定而非假設 |

## 與其他 domain 的關係

- **apa7-style**：管「寫作／引用格式」；本域管「報告／投影片交付」。A3（source fidelity）、A7（正確引用）與 apa7 的 citation 紀律互補、不矛盾。
- **note-writing / mathematical-writing**：同屬鄭澈的表達類 domain；本域專注口頭發表的即時性（mental load、時間 slot、navigation）。

## 用法

寫或審投影片／講稿時載入本域。A4（去 AI tell）與 colleague-che-cheng-academic 的「AI 八股黑名單」同源；A1/A5/A6 對應 colleague skill 的「Beamer 模式慣例（國際會議 oral）」段。
